CurveGPU Go WASM POC

This page compares two browser-side wasm paths on the same deterministic BN254 or BLS12-381 workloads: Go wasm calling the TS/WebGPU library via syscall/js, and pure Go wasm using gnark-crypto. The workload includes Fr NTT, G1 MSM, and G2 MSM to better approximate Groth16 prover hot paths. The WebGPU side performs one untimed prewarm pass before measured runs.
